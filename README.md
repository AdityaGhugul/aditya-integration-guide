# aditya-integration-guide
I spent 5 years in integration engineering.

3 of those years on Axway APIM at TCS, handling 100M+ API requests daily.

I knew OAuth flows. I had designed mTLS policies. I had debugged production incidents at 2am.

But in interviews, I kept stumbling.

Not because I didn't know the answers.

Because I couldn't say them in the language interviewers expected.

I'd say "checking the message to decide where to route it" instead of "content-based routing."

I'd say "stopping calls to a broken service" instead of "circuit breaker pattern."

The knowledge was there. The vocabulary wasn't.

So I spent 3 months building a complete interview guide:

→ 100 questions across 5 difficulty tiers (from instant-rejection basics to architect-level design)
→ 100 key terms with layman version, technical definition, AND the exact sentence to use in interviews
→ 10-week study plan that fits around a full-time job
→ Covers: API fundamentals, OAuth/JWT, integration patterns, messaging, observability, CI/CD

It's completely free. No email required. No paywall.

Link in first comment 👇

If this helps even one person explain their experience better in an interview, it was worth building.
